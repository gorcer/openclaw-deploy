# ТЗ: Сервис Нутрициологии (Telegram Бот)

## 1. Общее описание

**Название:** NutritionBot (или другое)
**Платформа:** Telegram бот
**Суть:** Человек проходит анкетирование → получает рекомендации по анализам → вносит результаты → получает заключение и 6-месячный курс витаминов.

---

## 2. Структура данных

### 2.1 Системы организма (10 штук)

| ID | Система | Описание |
|----|---------|----------|
| 1 | Нервная | Головной, спинной мозг и нервы. Управляет всеми процессами, обеспечивает мышление и реакции. |
| 2 | Опорно-двигательная | Кости, суставы, связки, мышцы. Обеспечивает опору, движение и защиту. |
| 3 | Сердечно-сосудистая | Сердце и сосуды. Кровообращение, транспорт кислорода и питательных веществ. |
| 4 | Дыхательная | Нос, глотка, гортань, трахея, бронхи, легкие. Кислород → CO2. |
| 5 | Пищеварительная | Рот, пищевод, желудок, кишечник, печень, поджелудочная. Расщепление и усвоение. |
| 6 | Эндокринная | Железы внутренней секреции (щитовидная, гипофиз и др.). Гормоны. |
| 7 | Мочевыделительная | Почки, мочеточники, мочевой пузырь. Вывод продуктов обмена. |
| 8 | Иммунная и лимфатическая | Лимфоузлы, сосуды, селезенка, костный мозг. Защита от инфекций. |
| 9 | Покровная | Кожа, волосы, ногти. Защита, терморегуляция. |
| 10 | Репродуктивная | Половые органы, размножение. |

**Каждая система = 7-10 симптомов.**

### 2.2 Справочники

- **Симптомы** — привязаны к системе, описание
- **Анализы** — название, норма, единицы измерения, где сдавать
- **Витамины/добавки** — название, состав, форма выпуска, дозировка, показания, противопоказания, как принимать
- **Правила симптом→анализы** — если отмечен симптом X, рекомендовать анализ Y
- **Правила интерпретации** — если анализ Y имеет значение Z, то заключение W
- **Правила витамины←(симптомы+анализы)** — на основе чего рекомендовать витамин
- **Совет директоров** — эксперты, чьими рекомендациями руководствуется интерпретация

## 3. Функционал

### 3.1 Анкетирование

**Этап 1: Знакомство**
- Приветственное сообщение
- Сбор имени, возраста, пола (опционально)

**Этап 2: Выбор системы → Симптомы**
- Пройтись по всем 10 системам
- Для каждой системы показать 7-10 симптомов с чекбоксами
- Человек отмечает беспокоящие симптомы
- Можно пропустить систему

**Этап 3: Рекомендация анализов**
- На основе отмеченных симптомов → список анализов
- Показать: название анализа, где сдавать, примерная стоимость (опционально)
- Возможность "У меня уже есть результаты" → переход к внесению

### 3.2 Внесение анализов

- Человек вводит результаты анализов
- Проверка формата (число, единицы измерения)
- Сохранение в профиль пользователя

### 3.3 Заключение

- **Отображение результатов** — таблица с значениями и нормой
- **Интерпретация** — текстовое заключение (например: "Гемоглобин понижен — возможна анемия. Рекомендуется...")
- **Источник рекомендаций** — "Совет директоров: [имена экспертов]"

### 3.4 Курс витаминов (6 месяцев)

**Логика:**
1. **Этап 1-2 (мес): Очищение + восстановление** — базовые витамины, очищающие
2. **Этап 3-4 (мес): Восстановление** — активное восстановление дефицитов
3. **Этап 5-6 (мес): Поддержание** — закрепление результата

**Рекомендации строятся на:**
- Симптомы (что беспокоит)
- Анализы (где дефицит)
- База витаминов (что подходит)

**Вывод:**
- Название витамина
- Состав
- Как принимать
- Продолжительность

## 4. Технические требования

### 4.1 Стек (предварительно)

- **Python** (Aiogram 3.x или Telegram Bot API)
- **База данных:** SQLite (для начала), или PostgreSQL
- **Хранение:** AWS S3 или локально (для базы витаминов, если файл)

### 4.2 Модули

1. **User Management** — регистрация, профиль, история
2. **Survey Engine** — анкетирование, сохранение симптомов
3. **Analyzer** — правила симптом→анализы, интерпретация анализов
4. **Vitamin Recommender** — алгоритм рекомендаций на основе симптомов+анализов
5. **Vitamin Base** — база витаминов (загрузка из Excel)
6. **Reports** — генерация заключений
7. **Admin Panel** (опционально) — управление справочниками

### 4.3 Интеграции

- Yandex SpeechKit (голосовые команды — опционально)
- Google Sheets / Excel import для базы витаминов
